# 前端协作与维护说明

这是黑客松 Demo 的展示前端。大家各自维护自己负责的页面区域和对应交互；修改前先同步，避免覆盖彼此的工作。

## 页面分工

- 01：CLI → Idea 的入口与项目介绍
- 02：CLI 能力清单
- 03：Idea 挖掘与选择
- 04：截图生成 App、CCCC Agent 运行日志与 Live MVP 展示

04 由截图生成 Worker 负责人维护。其他同学只修改自己负责区域的结构、样式和交互即可。

## 文件对应

- `index.html`：页面结构，按 01—04 区段定位
- `styles.css`：全局视觉与各区段样式
- `app.js`：页面交互与展示状态

## 协作约定

1. 从主分支新建自己的功能分支，例如 `feat/section-01`、`feat/section-02`。
2. 只改自己负责区段相关的 HTML、CSS 和 JavaScript。
3. 提交前在浏览器检查桌面端和手机端布局是否正常。
4. 通过 Pull Request 合并；如改动到公共样式或公共交互，先在群内同步。

GitHub Pages 可作为静态展示备份；比赛现场使用的真实运行服务由 DGX Spark 负责。
