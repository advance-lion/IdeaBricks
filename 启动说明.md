# 启动说明

## Windows 一键启动

1. 下载或 clone 本仓库并解压到本地目录。
2. 安装 Python 3.10 或更高版本；安装时勾选 **Add Python to PATH**。
3. 双击根目录的 [`启动投递台.cmd`](启动投递台.cmd)。
4. 等待浏览器自动打开 `http://127.0.0.1:4181/`；不要关闭弹出的命令窗口。

脚本只使用 Python 标准库，不需要先执行 `pip install`。关闭命令窗口即停止投递台。

## 启动后可以做什么

- 在投递台上传一张已获授权的截图。
- 在舞台页选择 **Codex CLI** 或 **Local LLM**。
- 生成期间查看截图、阶段进度；完成后查看可运行页面、预览图、验收报告和 GitHub 交付链接。

投递台可以独立启动；真正生成页面仍需要所选执行引擎可用：

- **Codex CLI（默认）**：在 PATH 中安装并登录 `codex`，或设置 `CODEX_BIN` 指向 `codex` 命令或 `codex.js`。
- **Local LLM**：在 `config/local-model.local.json` 填入本地 OpenAI 兼容服务的地址、模型与可选密钥。该文件已被 Git 忽略，不会上传。
- **CCCC**：从投递台立即派发 Worker 时需要本机已安装并登录 CCCC；未安装时仍可启动界面和创建契约，但无法派发。

## 常见问题

### 浏览器没有自动打开

手动访问 `http://127.0.0.1:4181/`。如果端口被占用，先关闭其他投递台窗口，或在命令行执行：

```bat
set MVP_PORT=4182
启动投递台.cmd
```

随后访问 `http://127.0.0.1:4182/`。

### 出现“未找到 Python 3”

安装 Python 3.10+ 后重新打开命令提示符，再双击启动脚本。也可以运行：

```bat
py -3 --version
```

确认 Python 已加入 PATH。

### 页面可以打开，但生成没有启动

先确认所选引擎可用：Codex CLI 需要 `codex --version` 能运行；Local LLM 需要有效的本地模型配置；点击“立即派发”还需要 CCCC 可用。投递台会保留错误信息和已创建的契约，便于修复后重试。
